Content Guide

Tips for Creating Effective Institute Advancement Pages

Content Checklist

  • Use snappy language in the title and subtitle and headings.
    • Consider active text that reinforces our mission and summarizes the text. For example, “Provide for our Platforms” rather than just “Platforms”.  The subtitle “Inquiry, Innovation, And Collaboration” adds additional meaning in a way that is also scannable.
  • Kick off a content rich page with a lead image (750px wide or bigger), don't use the lead image on a navigation focused grid page
  • Ensure the priority message of the main body content is accessible on small screens
  • Additional images, asides, pull quote, and headers can be placed with the main body content
  • The main body content should conclude with a call to action 
  • An aside can also be placed as a sidebar or below the main body
  • The page can conclude with linked grid boxes; all of the images in the grid should be a minimum of 620px and have the same aspect ratio

Make a Strategy

Write clear, succinct text that helps a user perform a specific action.

  • Have a clear goal for the page
    • User should read article? Hit a button?
    • The grid boxes emphasize navigation; main body content with inline images emphasizes content.
  • Consider the user experience
    • How did the user get to this page?
    • What are his/her assumptions about this page and what do they want to get out of it?
  • Create a strategy to incentivize the user to perform your intended action
  • Conclude your content with a call to action. What is the next thing you want the user to do?

Be Succint and Scannable

  • Less can be more
    • Too many choices can paralyze a user. 
    • Too much information can clutter a page, making clean design more challenging.
    • Too much content can distract from the priority.
  • Make scannable
    • Ensure content is useful for users who don’t read.
    • Can the user scan the page and still get information, meaning, and/or entertainment?
    • Break up blocks of text with images, pull quotes, asides, and headings.
  • Use storytelling to make it feel easy 
  • Consider page structure
    • From above the fold, how do you communicate the page sections and links?
    • There are many variations of page layouts. Check out the pattern library for some prebuilt options.

Additional Rules

  • iModules
    • Links for Sign Up and Log In have been combined. We will feature the Log In Page, which will include a prominent link for the Sign Up page.
    • For actions that require a log-in, we will emphasize a work flow where the log-in occurs in the middle of the path, rather than the beginning.
    • For example:  Homepage -> Event Description -> Register Button -> Log-in (if not already) -> Event Reg Details
    • User has the option to log in at any prior point as well.
  • For accessibility use image alt tags
  • Use descriptive link language instead of “click here”
