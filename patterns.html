Grids

These grids are the primary way to display multiple options. They provide the user with clear, scannable, choices of where to go next.

Drupal Instructions

Inline Images

Drupal Instructions

Lead Image

Create a lead image to insert an initial image onto the page as well as associating a page with a specific image. 

  1. Find the lead image section below the Aside.
  2. Upload Image.
  3. Optional: Fill in caption.

Additional Images

The Drupal CMS has a widget to help insert consistent images and captions. To access it:

  1. Create a new paragraph. The WYSIWYG will embed the image inside whatever container you are currently working in. Most of the time, you'll want a fresh parapraph to work inside.
  2. Then select the image widget at top. The widget is only visible when in Full HTML mode, which can be toggled just below the content field. 
  3. Upload or choose image.
  4. Fill out details, caption and title are optional.
  5. "Display As" formats the style. The style is also dependent on the Page Type.
  6. Do not select "Preview", "Teaser", or "WYSIWYG", they are pre-built defaults and will not look consistent.
  7. After the image is embedded, create a new paragraph to seperate the image from the next section of content. 
  8. To edit an image, click on the image in the WYSIWYG and then click on the image widget at top. 
  9. If deleting an image, remember to delete the extra paragraph you created to hold it in.
  10. Before saving the page, make sure you are in Full HTML mode, otherwise the image will not dispaly properly.

Image Sizes

The following are the recommended widths for images.

  • Grid: 620px
  • Aside: 415px
  • Lead image, full image, right image, symmetry: 750px
  • Small-left, small-right: 265px

Responsive Video

You can make your video embeds and iframes responsive. This pattern presets the aspect ratio, so you may need to add inline CSS to adjust for your video. Adjust the size by changing the class of the wrapper. See inline images/figures for options. Asides Within the Main Content

Think of the page content in three tiers of priority. Priority 3 is broken off into an aside, which can be displayed to the side or at the bottom - depending on the Page Type or template. Within the Main Content, Priority 2 is displayed off to the side of the Priority 1 content on larger screens; and on smaller screens the Priority 1 and Priority 2 are displayed as a single tube of content.

Drupal

In Drupal, create a div with class aside.

  1. On a new line, write aside text
  2. Select aside from the styles menu at top of WYSIWYG, wrapping text in a div

 

In iModules, where HTML5 elements are accepted, use an aside.

 

For asides that do not fit into the flow of content, add a class aside-callout. On small screens, these are displayed as a single tube and the callout will be differentiated.

Drupal

  1. On a new line, write aside text
  2. Select aside-call-out from the styles menu at top of WYSIWYG, wrapping text in a div

 

In iModules, where HTML5 elements are accepted, use an aside with class aside-breakout.

CSS Overwrite

iModules uses a combination of product-wide CSS, iModules custom generated CSS, and two client-created CSS files. They recommend avoiding using any ids in the CSS. The primary client created CSS file can be updated by emailing a request to Application Support. The three alumni templates have a client CSS and a quick fix CSS and the one give template has separate client CSS and quick fix CSS.

We also have access to an additional file for quick fixes. The live overwrites are called alumni-overwrite.css and giving-overwrite.css. This quick fix file is not branched for IE8 and all browsers will receive it. The file can be overwritten by using the document manager: /css/. There are separate quickfix files for Alumni and Give templates. The IA Website Development Summary describes the best practices for compiling and versioning the quickfix file. Prior to updating the file, make sure all other stakeholders are aware of the update.

  1. On an iModules served page, select a block. (You must be logged in with content admin access.)


  2. Select the Document Manager on the far right of the first row of the WYSIWYG.


  3. Select the css folder and then upload the updated CSS overwrite file.
  4. Rename the current overwrite file (if any) with a version number.
  5. Rename the new overwrite file with the appropriate name. Either alumni-overwrite.css or giving-overwrite.css.
